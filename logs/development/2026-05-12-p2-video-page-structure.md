## Target

把 Web UI 从“视频 / 仓库 / 创作者 / 队列 / 设置”的工具页集合，重构成一个本地优先的私人字幕库主页。主页只保留两个核心区域：上方收入库，下方浏览收藏夹；一级入口只保留主页和设置。

## Steps

1. 复核 `DESIGN.md`、用户效果图和现有前端文件。
2. 删除旧的五个顶层 Tab，改成可收起侧边栏：主页 / 设置。
3. 在主页上半部分实现“获取字幕”模块，支持视频 / 合集 / 来源三种模式。
4. 将前端默认日志改成人话进度步骤，原始日志继续保留在折叠区和后端任务日志目录中。
5. 在主页下半部分实现“收藏夹”模块，支持字幕 / 合集 / 来源三种浏览模式。
6. 重写设置页为收纳式页面，保留字幕库路径、ASR 默认值、Cookies 和重建索引入口。
7. 运行前端语法检查，并确认本地 UI 正常返回新主页。
8. 按人工复看反馈继续压缩“获取字幕”模块高度，减少内层方框感，并把收藏夹默认展示改为少量条目加“加载更多”。
9. 根据真实验收反馈修复：模式切换不再清空当前任务日志；详细日志改为浮层，不再撑高获取区；作业结果解析支持 stdout 前缀进度行。

## Changes

- `clipvault/ui/static/index.html`
  - 重建应用壳层：左侧侧边栏、主页、设置页。
  - 主页包含“获取字幕”和“收藏夹”两个大模块。
  - 获取字幕模块支持视频 / 合集 / 来源三模式。
  - 收藏夹模块支持字幕 / 合集 / 来源三模式。
- `clipvault/ui/static/style.css`
  - 按效果图重写布局，使用侧边栏、暖奶油背景、珊瑚主按钮、浅卡片和细边框。
  - 新增可收起侧边栏、人话进度步骤、收藏夹横向列表卡和响应式布局。
  - 调整第三轮视觉密度：获取区更矮，状态区改为轻分隔，收藏夹条目减少边框和阴影，高级选项箭头改为 CSS 细线图形。
  - 收起入口改成按钮样式，详细日志改为覆盖式面板，避免展开后把整个获取区撑长。
- `clipvault/ui/static/app.js`
  - 重写前端状态模型：主页 / 设置、获取模式、收藏夹模式、侧边栏收起、详细日志折叠。
  - 视频模式继续调用现有 `/api/video`，并通过 SSE 映射为人话状态步骤。
  - 来源模式复用 `/api/creators/add` 和 `/api/creators/fetch`。
  - 收藏夹从 `/api/library` 和 `/api/creators` 渲染字幕、合集与来源。
  - 收藏夹每个模式默认只展示 3 条，用户点击“加载更多”后每次增加 6 条。
  - 切换视频 / 合集 / 来源模式时只切换表单文案，不再清除当前运行日志。
- `clipvault/ui/server.py`
  - Web 作业成功时从 stdout 末尾提取 JSON 结果，兼容 yt-dlp 进度行污染 stdout 的情况。
- `tests/test_ui_server.py`
  - 新增 stdout 前缀进度行后仍能解析最终 JSON 的回归测试。
